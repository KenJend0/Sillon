1. POSITIONNEMENT VISUEL
───────────────────────

Sillon est :

  Intime — un journal personnel, pas un réseau social
  Éditorial — le contenu prime sur l'interface
  Analogique — la chaleur du papier, pas le froid du numérique
  Silencieux — l'UI s'efface derrière la musique et les mots

Ce n'est pas :

  Un réseau social demonstratif
  Une app de streaming colorée
  Une interface gaming ou techno
  Une app d'avis / notation froide

Tout doit respirer. Si l'UI attire l'attention, c'est trop.

Test final : une capture en noir et blanc reste lisible.
             une page imprimée fonctionne.
             l'interface disparaît mentalement après 3 secondes.

3. TYPOGRAPHIE
──────────────

  Sillon est éditorial. Deux polices, deux rôles distincts.

  POLICES
  ────────
  Instrument Serif 400   → Titres, display, identité
  Inter 400/500          → Corps, labels, fonctionnel

  Instrument Serif italique réservé à :
    • Mots d'emphase dans les titres (ex: "Pour toi", "cette semaine")
    • Chiffres des badges de note (9/10)
    • Chiffres des stats du profil (55 critiques)
    • Citations entre guillemets « »
    • Comptes dans les listes (17 items)
    • Labels de dates dans les séparateurs (Hier, 15 mai)

  HIÉRARCHIE
  ───────────
  h1 — Instrument Serif 400 / 32px / line-height 1.1 / letter-spacing -0.01em
       → Titre de page (Explorer, Feed, Mon profil)

  h2 — Instrument Serif 400 / 22px / line-height 1.2 / letter-spacing -0.005em
       → Titre de section (Pour toi, Tendances, Critiques)
       ! Les h2 fonctionnels (modales, sheets) restent en Inter via font-sans

  h3 — Inter 500 / 24px → Sous-sections fonctionnelles

  body — Inter 400 / 16px / line-height 1.75
  meta — Inter 400 / 14px / line-height 1.5
  sm   — Inter 400 / 13px / line-height 1.5    (titres de cards en grille)
  label — Inter 500 / 12px / letter-spacing 0.06em

  EYEBROW (pattern récurrent)
  ────────────────────────────
  Inter 11px / uppercase / tracking-[0.22em] / text-tertiary
  Usage : étiquettes de section compactes, axes d'histogramme, colonnes de stats

  NOMS D'ALBUMS & TRACKS (dans les cards)
  ────────────────────────────────────────
  font-display (Instrument Serif) / font-normal / text-sm (13px)
  Couleur : text-warm (#2A2520)
  Toujours line-clamp-2, jamais truncate sur une seule ligne

  CITATIONS (extraits de critiques)
  ───────────────────────────────────
  Inter italic / text-meta (14px) / leading-relaxed
  Avec guillemets français : « {texte} »
  Couleur : text-secondary

  Jamais Bold 700. Jamais de mix de polices non documenté.


4. ESPACEMENT
─────────────

  Base unit = 4px

  section-lg   48px   Entre grandes sections de page
  section-md   32px   Entre sections moyennes
  section-sm   16px   Entre éléments d'une section
  micro        8px    Gap minimal

  Max-width contenu : 672px (max-w-page)
  Padding horizontal : px-4 mobile / px-6 md / px-8 lg

  Règle : si un bloc semble vide → augmenter le padding, pas la décoration.


5. CARTES & SURFACES
────────────────────

  CARD STANDARD
  ──────────────
  Background : bg-secondary (#ECE8E1)
  Border     : 1px solid border (#D8D3CB)
  Radius     : 12px
  Shadow     : aucune ou subtle (0 1px 2px rgba(0,0,0,0.04))

  CARD PAPIER (sidebar, banners, zones d'emphase)
  ─────────────────────────────────────────────────
  Background : paper-hi (#FAF8F4)
  Border     : 1px solid border
  Radius     : 14px
  Barre accent gauche optionnelle : w-0.5 bg-accent opacity-40 (filet d'identité)

  FEED CARDS (activité)
  ──────────────────────
  Background : bg-tertiary (#E4DFD6)
  Radius     : 12px
  Barre accent gauche : absolute left-0 top-6 bottom-6, w-0.5, accent, opacity-40

  Pas de gradient. Pas de glow. Pas de fond accent massif.

  SUIVRE / SAGE
  ──────────────
  Border : 1px solid sage (#7A8471) / Text : sage / Radius : 99px

  ─────────────────────────────────────────────────
  Background : transparent / Border : 1px solid accent (#8E6F5E)

  Padding : 6px 12px / Font : Inter 500 / 11.5–13px
  Hover : bg-accent text-paper-hi
  Désactivé : border-border text-tertiary cursor-default
  Usage : "Suivre" (goûts similaires), actions d'engagement secondaire,

  Jamais de bouton énorme. Jamais d'accent en fond sur plus de 50% de la largeur.


9. NAVIGATION MOBILE (bottom nav)
──────────────────────────────────

  Layout    : pilule flottante, left-4 right-4 bottom-4
  Background: paper-hi/90 + backdrop-blur-md
  Border    : 1px solid border / Radius : 22px
  Shadow    : 0 8px 20px -8px rgba(60,40,20,0.18)

  Onglet actif : filet accent 2px en haut de l'icône + label text-accent-deep
  Onglet inactif : text-tertiary

  Bouton AJOUTER (signature de l'app) :
    Carré sombre (bg-text-warm) / rounded-[10px] / rotate(-4deg)
    Icône en paper-hi / shadow warm

  Avatar (onglet Moi) actif : ring-2 accent autour de l'avatar


10. IMAGES & COVERS
──────────────────

  Grilles (explore, journal) : ratio 1:1 / radius 10px
  Cards compactes (feed, search) : 80×80px / radius 10px
  Cards très compactes (sidebar) : 36×36px / radius 5px
  Covers dans les listes (collage 2×2) : radius 8px au global

  Pas d'ombre dramatique. Filet intérieur subtil : inset 0 0 0 1px rgba(0,0,0,0.04)


11. ICÔNES
──────────

  Taille standard : 16–20px (Lucide React)
  Stroke : 1.5–1.6
  Couleur par défaut : text-tertiary
  Hover : text-secondary ou text-accent selon contexte
  Actif (like, save) : fill-like ou fill-accent


12. INPUTS
──────────

  Background : bg-secondary / Border : border neutre
  Radius : 10px / Padding : 12px / Text : principal
  Placeholder : tertiary
  Focus : border accent, outline none
  Font-size : 16px minimum (évite le zoom iOS)


13. INTERACTIONS
───────────────

  Durées : 150ms (hover colors) / 200ms (opacity, transform) / 300ms (modales)
  Easing : ease-out standard
  Transitions autorisées : colors, opacity, transform (translateY max 4px)

  Jamais : bounce / scale violent / animation > 300ms / flash coloré


14. STATS CHIFFRÉES (profil)
────────────────────────────

  Chiffre : Instrument Serif italic / 28px / text-warm
  Label   : Inter / text-label / uppercase / tracking-[0.14em] / text-tertiary
  Layout  : flex, chaque stat flex-1, séparés par filets verticaux (border-r rule)
  Filet horizontal supérieur : border-t rule avant le bloc de stats


15. TOASTS
──────────

  Composant : components/Toast.tsx
  Fonction  : showToast(message, type)

  success → fond #1C1C1C / texte #F5F3EF / pas d'icône
  info    → fond #1C1C1C / texte #F5F3EF / pas d'icône
  error   → fond #1C1C1C / texte #C86C6C / AlertCircle

  Durée : 4 secondes / Position : bottom-20 mobile, bottom-4 right-4 desktop
  showToast est client-only — appeler depuis un handler client après await.
  <Toast /> monté une seule fois dans le layout global.


16. CE QUI EST INTERDIT
────────────────────────

  ❌ Blanc pur (#FFFFFF) en fond
  ❌ Noir pur en texte
  ❌ Gris froid ou bleuté
  ❌ Vert flashy, bleu vibrant, couleur saturée
  ❌ Gradient décoratif
  ❌ Shadow lourde ou glow
  ❌ Border radius incohérent (toujours depuis les tokens)
  ❌ Mix de polices non documenté (Inter + Instrument Serif uniquement)
  ❌ Instrument Serif sur du texte courant (corps, labels)
  ❌ Bold 700
  ❌ Animation > 300ms ou bounce
  ❌ Accent en fond massif sur de grandes zones
  ❌ Bouton énorme ou trop coloré
  ❌ Interface qui attire plus l'attention que le contenu
